# Fix: menu selezione tipologia documento

## Cosa è cambiato
Il menu bottom-sheet che appare dopo il caricamento di un documento aveva le etichette delle categorie rese con `font-size: 26px` ereditato dal pulsante (pensato solo per le emoji). Risultato: testo enorme, layout sgangherato su mobile.

### Correzioni applicate (`styles.css`)
- `.doc-cat-btn`: aumentata l'emoji a `30px` con `line-height: 1` per renderla pulita
- `.doc-cat-btn span`: aggiunto stile esplicito `font-size: 12px; font-weight: 500; color: var(--text-2)` — separato dal font dell'emoji
- `.doc-modal-box`: bottom padding aggiornato a `max(calc(env(safe-area-inset-bottom) + 20px), 32px)` per rispettare la home indicator su iPhone
- `.doc-modal-box`: `max-width` portato da 420px a 480px; aggiunto `box-sizing: border-box`

## File modificati
- `styles.css` (regole `.doc-cat-btn`, `.doc-cat-btn span`, `.doc-modal-box`)
