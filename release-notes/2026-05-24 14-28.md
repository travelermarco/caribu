# Heater BLE fix — complete rewrite

## Cosa è cambiato
Il riscaldatore diesel non rispondeva a nessun comando. Riscritto completamente `heater.js`.

## Bug corretti
1. **Caratteristica di scrittura sbagliata** — `_connectDevice()` ora enumera TUTTE le caratteristiche con `getCharacteristics()` e seleziona separatamente quella con `notify` (per ricevere dati) e quella con `write`/`writeWithoutResponse` (per inviare comandi). I cloni HM-10 economici usano FFE1 per notify e FFE2 per write.
2. **Logica fallback invertita in `_send()`** — il vecchio codice nel catch tentava `writeValueWithResponse` quando la proprietà era `writeWithoutResponse=true` (inverso). Ora prova sempre `writeValueWithoutResponse` prima (nessun controllo di proprietà), poi fallback a `writeValueWithResponse`.
3. **`_parse()` ignorava frame corti** — cambiata la soglia minima da 7 a 3 byte, aggiornando solo i campi presenti. Aggiunta validazione header `[0xAA, 0x55]`.
4. **Nessuna visibilità sui dati ricevuti** — aggiunto log console `Heater RX ←` e campo `rawHex` nella UI (card Info).
5. **Poll troppo frequente** — da 3s a 5s; poll manuale dopo ogni comando anziché timer fisso.

## File modificati
- `js/heater.js` — riscritta da zero
- `js/app.js` — aggiunto `rawHex` allo stato heater, visualizzazione debug nella card Info
- `sw.js` — versione cache bumped da `caribu-v5` a `caribu-v6`
